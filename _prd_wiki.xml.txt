<title>智能物料审核面试稿</title>
<p>项目时间20251月开始进行需求调研分析，3月份开始启动开发，分为3期，一期从3月到5月搭建MVP，实现基础流程，实现了文本审核。二期从5月到7月迭代优化并实现了指标校对和合规规则审核。三期从8月到10月持续算法迭代优化，后续逐步拓展需求场景，由原本的单一文案审核，逐步添加出图文比对，图图比对等功能；</p>
<h2>一、项目背景</h2>
<p>经了解公司在对外营销宣传海报的审核中存在合规审查效率低、漏检、审核规则复杂等问题，因此希望通过搭建AI智能审核实现营销物料的智能解析、识别、数据比对和合规规则检验，从而提升审核效率，减少合规问题。</p>
<p>时间周期：2025.1~2026.3，整体做了1年左右的项目落地与迭代。</p>
<hr/>
<h2>二、前提准备工作</h2>
<p>1.分析业务需求，与需求方探讨如何在他们实际的业务流程场景中，赋予AI的能力。我梳理出了他们在物料审核这一块的工作流程，以及拆解了在这个流程中，不同角色如营销人员、合规人员、设计师的工作流程。最终决定先在合规审核人员这一环节中引入智能审核，增强审核效率，后续再逐步拓展其它角色的业务场景，甚至打通全流程，与OA等系统打通闭环；</p>
<p>2.紧接着和技术团队探讨技术方案，决定是采取多智能体协同的方式，dify+workflow编排+rag自研检索，而且后续这些智能体也能复制到其它项目中使用，可拓展性强。模型计划采用qwen3-32B私有化大模型+lora微调和qwen2.5VL模型；同时技术团队和我们一起先进行了可行性验证；</p>
<p>3.数据收集（知识工程）：共同确认数据的收集与标注工作，对历史人工审核物料、合规规则等数据做准备，整理出标准审核QA、合规规则清单。</p>
<hr/>
<h2>三、1.0版本：MVP产品（2025.4~2025.6）</h2>
<p>
<b>核心指标：</b>
</p>
<ul>
<li>审核准确率：≥95%</li>
<li>平均响应时间：≤500ms/材料</li>
<li>人工介入率：≤20%</li>
<li>系统可用性：≥99%</li>
</ul>
<p>交互界面：依据对需求和业务流程的梳理，用户界面功能点包括：用户上传界面，审核界面页面，审核疑点标注，人工核对（收集核对情况进行调优）。</p>
<p>dify+workflow编排+Rag自研，实现对营销物料的文本校验；</p>
<p>节点包括：多模态大模型识别、实体识别智能体、文本校对智能体（规则库）</p>
<p>
</p>
<hr/>
<h2>四、2.0版本：核心迭代，降本+提效+合规性双突破（2025.6~2026.3）</h2>
<p>1.0上线后，核心目标转向<b>提升审核准确率、拓展场景</b>
</p>
<p>核心指标</p>
<ul>
<li>审核准确率：≥98.5%</li>
<li>平均响应时间：≤100ms/材料</li>
<li>人工介入率：≤10%</li>
<li>系统可用性：≥99.9%</li>
</ul>
<h3>（一）拓展场景</h3>
<p>增加OCR识别、指标校对智能体（对接odp数据平台）、合规规则智能体（规则库、公司内规与外规）、错误解析智能体</p>
<h3>（一）性能、界面优化：</h3>
<p>技术上实现对图片审核的整体速度流程的优化。从单张图片20分钟，缩短至40分钟。还有界面上的功能优化；增加错误坐标标注等；</p>
<h3>（二）多模态模型幻觉</h3>
<ul>
<li>多模态大模型经常会出现幻觉；我们收集了badcase，进行针对性优化；同时我们还增加多了一个ocr识别，让ocr与多模态大模型共同对图片进行解析；有效减少模型幻觉；</li>
</ul>
<h3>(三)上下文工程</h3>
<p>针对实体识别智能体，我们拓展了合规校验和指标校验智能体之外，需要将原本独立 散乱的识别出来的实体 组装成结构化信息，这是进行指标和合规校验的基础；图片中关于某个基金的相关信息的布局是不固定的，输出的md文件也一样。因此我们需要实体识别智能体能逐一输出xx基金的代码是xxx，xx基金截止xxx日的累计收益是xx。也要能够将表格的数据指标读取成xxx基金xxx日的业绩数据是xxx。整理出可审核语句后，再注入后续智能体中；</p>
<p>
</p>
<h3>（五）知识工程与知识库体系优化</h3>
<ol>
<li seq="1">一方面补充知识来源，从历史人工审核案例中挖掘隐藏的审核规则、高频错误场景，结合监管合规规则、基金备案数据，构建结构化知识库体系，将基金核心实体、合规规则、高频错误案例分开存储；</li>
<li>会有规则冲突的情况：需建立规则优先级体系；定期规则评审</li>
<li>RAG调优：如每个切割后的 “单元”（Chunk）必须是一条 “完整且独立的合规要求”，能单独支撑一次检索匹配。要找到切割大小和信息密度平衡点；</li>
<li>规则引擎（独立于RAG）：检索召回知识片段后，先送入规则引擎对片段做实体切割与提存再送入校验智能体。例子：知识库中提到xxx基金的基金跟踪指数收益率xxx；大模型可能会把后半段收益公式也当成跟踪指数的一部分，然后和海报上提取到的实体一对比，就误判了。所以可以用规则引擎告诉切割好，能有效减少幻觉；当然你说用promt告诉大模型这个规则也可以，但在金融场景，还是会优先选择更稳定的方法。比如prompt也不是万无一失的；</li>
</ol>
<p>
</p>
<h3>（六）评测体系与数据飞轮迭代</h3>
<p>基于1.0已建立的评测机制，进一步完善线上线下数据闭环：</p>
<ul>
<li>线上：通过审核数据采集、案例分类、人工复核，反向优化知识库、提示词与召回策略，用真实线上数据驱动系统持续迭代；</li>
<li>离线：定期用历史审核数据测试模型性能，监控准确率、召回率、误判率、漏判率等指标，形成持续优化的闭环，让审核准确率与合规性稳步提升。</li>
</ul>
<hr/>
<p>
</p>
<h2>五、后续规划（3.0版本）</h2>
<p>当前系统核心能力已完成落地，后续规划向全工程化架构演进，进一步提升系统性能、稳定性与可控性’</p>
<p>拓展更多场景，增加视频的审核等；</p>
<h3>后续思考：业务流程优化：多场景审核链路分层拆分</h3>
<p>为进一步提升审核准确率，2.0版本对审核流程做了分层拆分：</p>
<ul>
<li>针对「海报物料」和「纯文本文案」两大场景，设计独立审核链路：海报物料走Qwen2.5VL解析+Qwen3 Max校验的流程，纯文本文案走轻量化文本模型校验，避免复杂多模态模型的不必要调用；</li>
</ul>

<h2>六、个人核心贡献</h2>
<ol>
<li seq="1">主导1.0版本快速落地，验证AI合规审核的可行性，同步搭建基础评测体系；</li>
<li seq="2">核心推进2.0降本优化，完成关键的模型替换与节点拆分，实现审核成本大幅下降；</li>
<li seq="3">解决模型幻觉、实体误判、知识库匹配错误等核心问题，设计实体边界规则+结构化知识库的优化方案；</li>
<li seq="4">决策审核流程分层拆分，优化多场景物料的审核链路，适配不同业务场景需求；</li>
<li seq="5">负责上下文工程、知识工程全流程落地，搭建结构化知识库与知识优先级规则；</li>
<li seq="6">完善数据飞轮迭代机制，通过线上线下评测持续优化系统，提升审核准确率、降低漏判率。</li>
</ol>

<h2>七、项目成果</h2>
<ol>
<li seq="1">完成模型替换与流程优化，实现智能合规审核服务成本大幅降低，同时物料处理效率显著提升；</li>
<li seq="2">审核准确率显著提升，新增复杂海报、长文案的审核能力，覆盖全场景基金宣传物料审核；</li>
<li seq="3">解决模型幻觉、实体误判、知识冲突等核心问题，审核漏判率、误判率大幅下降；</li>
<li seq="4">搭建了完善的上下文、知识工程体系与评测迭代闭环，系统具备持续优化能力。</li>
</ol>
