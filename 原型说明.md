# 智审 · AI 智能图片审核系统 — 原型说明（MVP & V1）

> 来源 PRD：飞书《AI 智能图片审核系统 PRD（MVP & V1 版本）》 `YpmYdGnGCoRO3kxks7AcJYrvnQb`
> 竞品参考：Calliper（calliper.cn，鉴权方式为微信扫码登录，原型登录页已纳入「扫码登录」选项）
> 原型文件：`ai-image-review-prototype.html`（单文件、可直接浏览器打开，侧边栏切换 7 个页面）

## 一、原型概述

### 核心假设（需验证）
- **假设 1**：金融行业营销材料（基金/理财海报、文档、H5）的合规审核，可由「多智能体协同 + 规则引擎」自动完成主体判断，将人工介入率从 100% 降至 ≤20%（MVP）。
- **假设 2**：审核员愿意把 AI 当作「助手」而非被替代——人机协作闭环（AI 初审 → 低置信度转人工 → 人工修正 100% 回流）能持续提升准确率。
- **假设 3**：规则库热更新 + RAG 知识库，能应对法规频繁变化，避免系统失效。

### 成功度量（取自 PRD）
| 指标 | MVP 目标 | V1 目标 |
|---|---|---|
| 审核准确率 | ≥95% | ≥98.5% |
| 平均响应时间 | ≤500ms | ≤100ms |
| 人工介入率 | ≤20% | ≤10% |
| 检查项覆盖率 | 58% (32/55) | 100% (75) |

### 范围与取舍（Speed-First）
- **MVP 纳入**：图片(PNG/JPG) + 文档(DOCX) 审核；32 条检查项（文本18/指标2/合规12）；人工复核队列；规则库；评测看板。
- **V1 预留（原型占位）**：H5 页面、视频（试点）；检查项扩至 75 条；云端 API 模型切换。
- **暂不构建**：模型训练后台、知识图谱编辑、多租户管理——属于算法/平台层，非 MVP 验证重点。

## 二、信息架构与页面清单（7 屏）

```
登录页 (账号 / 扫码[参考Calliper])
└─ 工作台 Dashboard
   ├─ 审核任务 Tasks（列表 + 筛选 + 查看）
   ├─ 审核详情 Review（图标注 + 分类检查结果 + 决策栏）★核心屏
   ├─ 人工复核 Human（队列 + 复核面板 + 意见回流）
   ├─ 规则库 Rules（分类树 + 规则表 + 启停）
   └─ 评测中心 Eval（准确率指标 + 模型对比 + Badcase/Goodcase 管理）
```

### 各屏要点
1. **登录页**：账号登录 + 扫码登录双 Tab（扫码 Tab 直接对齐竞品 Calliper 的微信扫码鉴权体验）。
2. **工作台**：4 个 KPI 卡（今日审核量 / 准确率 / 人工介入率 / 响应时间）+ 7 日趋势折线 + 结论分布环图 + 各智能体达标率条形图。
3. **审核任务**：AntD 风格表格，状态标签（审核中/自动通过/自动驳回/人工复核），「查看」跳转审核详情。
4. **审核详情（核心）**：左侧模拟基金海报 + 4 处错误标注框（禁用词/缺基准/图表无来源/缺风险提示），右侧按「文本校对/指标校对/合规规则」分组列出命中项，底部置信度环（76% < 80% 触发转人工）+ 通过/驳回/转人工决策按钮。
5. **人工复核**：左侧优先级队列（置信度/高风险/异常冻结），右侧复核面板含 AI 原始结论对比、复核结论按钮、复核意见（标注 100% 回流）。
6. **规则库**：左侧分类树（文本48/指标22/合规5），右侧规则表含严重级别、启停状态、命中次数。
7. **评测中心**：准确率指标卡 + 模型性能对比表（Qwen-VL-7B / Max / DeepSeek / GPT-4V）+ 数据回流闭环 + Badcase/Goodcase 管理表。

## 三、验证框架

### A/B 测试设想
- **决策按钮排版**：「通过 / 驳回 / 转人工」三按钮的视觉权重顺序 A/B，观察人工复核误操作率。
- **置信度阈值**：对比 80% vs 85% 阈值的「人工介入率 / 漏检率」权衡，用看板数据反推最优阈值。

### 反馈收集（Day 1 起内置）
- 审核详情页决策栏记录每次人工动作 + 理由；人工复核意见 100% 回流（对应 PRD 6.3 数据闭环）。
- 评测中心 Badcase 表直接作为迭代输入，闭环：发现 → 根因 → 修复 → 沉淀 Goodcase 回归。

### 迭代节奏（对齐 PRD 5.4）
- 小版本每 2 周（Bug/规则优化）→ 中版本每月（新检查项）→ 大版本每季（架构/大模型）。

## 四、技术栈对齐（PRD 3.4）
前端 React + Ant Design（本原型用纯 HTML/CSS 模拟其视觉语言，零构建即可预览）；后端 Spring Boot + FastAPI；向量库 Milvus；规则引擎 Drools；存储 MySQL+MongoDB+MinIO；部署 K8s。
